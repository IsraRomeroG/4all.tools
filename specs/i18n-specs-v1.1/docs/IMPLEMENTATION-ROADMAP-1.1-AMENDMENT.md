## 2. Corrected cross-phase handoffs

### P00 → P01

P00 configuration smoke tests MUST support the migration:

```text
astro.config.mjs
    ↓
astro.config.ts
```

P01-T01 owns the migration and test update.

### P02 → P04

P02 exports taxonomy status. P04 enforces that every route requiring a taxonomy path has a fully published root-to-primary-category chain.

### P04 → P06

P04 owns route DTOs. P06 MUST connect feature route metadata through:

```text
src/routing/providers/tool-route-provider.ts
```

P06 domain contracts MUST NOT import P04.

### P05 → P06

P05 freezes `ToolPageModel.presentation` as required. P06 supplies the first production presentation provider at:

```text
src/templates/page-models/providers/tool-presentation-provider.ts
```

### P06 feature aggregation

Production aggregation lives under:

```text
src/features/tools/
```

not under `src/domain/tools/`.

## 6. Updated stop-the-line conditions

In addition to the original roadmap conditions, implementation MUST stop when:

- a `src/domain/` file needs to import a feature, `.astro`, routing, or delivery module;
- P06 attempts to import P04 route contracts into pure domain tool types;
- a generic landing target is required before an owning phase exists;
- a tool route would traverse a draft or archived taxonomy ancestor;
- two modules independently own the site origin;
- `/en/` can be generated by dynamic entity routing;
- `ToolPageModel.presentation` is missing or optional.
